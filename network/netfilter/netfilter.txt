🧱 What is Netfilter

Netfilter is a packet filtering framework built into the Linux kernel.
It provides hooks inside the network stack that allow kernel modules to inspect, modify, accept, drop, or redirect packets.

You can think of it as the engine behind:

🔥 iptables

🧱 nftables

🐳 Docker’s built-in NAT and bridge networking

🔐 Linux firewalls, NAT gateways, connection tracking, etc.




1. The Core Architecture: The Five Hooks
Netfilter works by placing "hooks" at specific points in the networking stack. When a packet reaches one of these points, the kernel pauses and asks the Netfilter rules: "What should I do with this packet?"

PREROUTING: Triggered as soon as a packet enters the network interface, before a routing decision is made. (Ideal for Destination NAT).

INPUT: Triggered if the packet is destined for a process running on the local machine.

FORWARD: Triggered if the packet is just passing through (e.g., your machine is acting as a router).

OUTPUT: Triggered for packets generated by the local machine.

POSTROUTING: Triggered just before the packet leaves the network interface. (Ideal for Source NAT/Masquerading).

2. Packet Filtering (The "Filter" Table)
This is the most common use of Netfilter. It decides whether to let a packet pass based on rules.

ACCEPT: Let the packet through.

DROP: Silently ignore the packet (sender gets a timeout).

REJECT: Stop the packet and send an error back to the sender (ICMP Destination Unreachable).

3. NAT (Network Address Translation)
Netfilter handles how IP addresses are rewritten as they move through the system.

Source NAT (SNAT): Changes the source IP of a packet. Typically used when a private network shares one public IP to reach the internet.

Destination NAT (DNAT): Changes the destination IP. Used for "Port Forwarding" to send outside traffic to a specific internal server.

Masquerading: A special form of SNAT for dynamic IP addresses (like a home router).

4. The Evolution: From iptables to nftables
The Linux kernel has migrated to a newer system called nftables, though the concepts remain the same.
