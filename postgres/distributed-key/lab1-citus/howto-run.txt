4. Running the Distribution Test
Now, let's create a table and see how the Distributed Key affects where data goes.

Step A: Create a standard table


CREATE TABLE sensor_data (
    sensor_id int,
    site_id int,
    reading float,
    recorded_at timestamp DEFAULT now()
);



Step B: Distribute by site_id
This is the "Distributed Key." Citus will use this column to decide which worker gets which row.



SELECT create_distributed_table('sensor_data', 'site_id');



Step C: Insert test data


INSERT INTO sensor_data (sensor_id, site_id, reading) VALUES (1, 1, 23.5); -- Should go to one worker
INSERT INTO sensor_data (sensor_id, site_id, reading) VALUES (2, 10, 18.2); -- Should go to the other


5. Verify the Distribution
To see exactly how the rows are split across your Docker containers, run:

SELECT site_id, count(*) 
FROM sensor_data 
GROUP BY site_id;


If you chose a good key, you should see the row counts split between worker-1 and worker-2.
