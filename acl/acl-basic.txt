ACL (Access Control List) in Linux is an extended permission system that allows you to give fine-grained permissions to files and directories beyond the standard owner / group / others model.

Most distros (Ubuntu, Debian, CentOS, Rocky, AlmaLinux, Fedora) no longer show acl explicitly in the mount options, because it is the default kernel behavior.




Linux normally supports only 3 permission sets:

| Entity | Permissions |
| ------ | ----------- |
| Owner  | rwx         |
| Group  | rwx         |
| Others | rwx         |





But what if you want:

give user Alice read access

give user Bob write access

give group testers read/write


while keeping the default owner/group permissions?

â†’ This is where POSIX ACL helps.

ACL allows you to assign permissions per user and per group, in addition to default permissions.













touch test.txt
-rw-r--r-- 1 root root 0 Nov 16 12:42 test.txt





setfacl -m u:username:rw test.txt


ls -l will show a + if ACL exists:

-rw-rw-r--+ 1 root root 0 Nov 16 12:42 test.txt


The + means: this file has extended ACL entries.





getfacl test.txt
