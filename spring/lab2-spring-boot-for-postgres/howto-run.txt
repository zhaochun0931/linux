1. Using the Maven Wrapper (Recommended)

The project generated by the CLI includes a "wrapper" (mvnw). This is the best way to run it because it ensures you use the correct Maven version without needing to install Maven on your Mac.

Run these commands in your project root:

# 1. Make the wrapper executable (only needed the first time)
chmod +x mvnw

# 2. Compile and run the app
./mvnw spring-boot:run




2. Running the Compiled JAR

If you want to simulate how the app would run on your bare-metal server, you should package it into a single "Fat JAR" file first.

# 1. Package the application (skips tests to save time)
./mvnw clean package -DskipTests

# 2. Run the resulting JAR file
java -jar target/million-data-batch-0.0.1-SNAPSHOT.jar









# Start the 1 million row insert
curl -X POST http://localhost:8080/batch/run

# Check the progress every few seconds
curl http://localhost:8080/batch/status
