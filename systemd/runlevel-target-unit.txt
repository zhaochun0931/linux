Previous versions of Red Hat Enterprise Linux, which were distributed with SysV init or Upstart, implemented a predefined set of runlevels that represented specific modes of operation. These runlevels were numbered from 0 to 6 and were defined by a selection of system services to be run when a particular runlevel was enabled by the system administrator. 

In Red Hat Enterprise Linux 7, the concept of runlevels has been replaced with systemd targets . Systemd targets are represented by target units. Target units end with the .target file extension and their only purpose is to group together other systemd units through a chain of dependencies.







üß† 1. The Old Way ‚Äî Runlevels (SysVinit)

In classic SysV-based Linux systems (e.g., old Ubuntu, CentOS 6):

The system booted into a runlevel, a number representing a system state.

Each runlevel defined which services were running (by executing scripts in /etc/rc.d/rcN.d/).

| Runlevel | Meaning                      | Typical State                    |
| -------- | ---------------------------- | -------------------------------- |
| 0        | Halt                         | System shutdown                  |
| 1        | Single-user mode             | Maintenance shell, no networking |
| 2        | Multi-user (no NFS)          | Often default in Debian          |
| 3        | Multi-user (full networking) | Text mode login                  |
| 4        | Unused                       | Custom                           |
| 5        | Graphical                    | GUI login (X11, GDM, etc.)       |
| 6        | Reboot                       | Restart system                   |








‚öôÔ∏è 2. The Modern Way ‚Äî Target Units (systemd)

Systemd replaced runlevels with target units, which are special kinds of systemd units that group services together to reach a desired system state.

A target = ‚Äúa collection of units‚Äù (services, sockets, mounts, etc.)

Example: multi-user.target replaces ‚Äúrunlevel 3‚Äù

| Target              | Replaces Runlevel | Purpose                                                                                  |
| ------------------- | ----------------- | ---------------------------------------------------------------------------------------- |
| `poweroff.target`   | 0                 | Shut down                                                                                |
| `rescue.target`     | 1                 | Single-user mode                                                                         |
| `multi-user.target` | 3                 | Multi-user, no GUI                                                                       |
| `graphical.target`  | 5                 | Multi-user + GUI                                                                         |
| `reboot.target`     | 6                 | Restart system                                                                           |
| `default.target`    | ‚Äî                 | The default boot target (usually a symlink to `multi-user.target` or `graphical.target`) |






üí° 6. Summary

Runlevels = old, numeric system states in SysVinit
Targets = modern, named, flexible dependency groups in systemd







üß≠ Runlevels vs. Systemd Targets

| **Runlevel** | **Systemd Target**                | **Description**                                          | **Typical Use Case**                  | **Command (systemd)**                 |
| :----------: | :-------------------------------- | :------------------------------------------------------- | :------------------------------------ | :------------------------------------ |
|     **0**    | `poweroff.target`                 | System halt / shutdown                                   | Powering off the machine              | `systemctl isolate poweroff.target`   |
|     **1**    | `rescue.target`                   | Single-user mode, minimal environment, maintenance shell | Troubleshooting / recovery            | `systemctl isolate rescue.target`     |
|     **2**    | `multi-user.target` *(Debian)*    | Multi-user, non-graphical (network + basic services)     | Servers, text-only environment        | `systemctl isolate multi-user.target` |
|     **3**    | `multi-user.target`               | Multi-user mode with networking                          | Common for headless servers           | `systemctl isolate multi-user.target` |
|     **4**    | *(custom)* or `multi-user.target` | Undefined / user-defined                                 | Rarely used; can define custom target | `systemctl isolate custom.target`     |
|     **5**    | `graphical.target`                | Multi-user + graphical login (X11 / Wayland)             | Desktop environments                  | `systemctl isolate graphical.target`  |
|     **6**    | `reboot.target`                   | Reboot system                                            | Restarting the system                 | `systemctl isolate reboot.target`     |
